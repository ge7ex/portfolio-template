2. ระบบแบ่งเป็น 4 ชั้น

Data Layer — ชั้นข้อมูล

เก็บสิ่งที่ผู้ใช้กรอก เช่น ชื่อ ตำแหน่ง ทักษะ ผลงาน รูป ภาษา และธีม

หลักที่ดี: แยกข้อมูลไทย/อังกฤษชัดเจน และไม่เปลี่ยนชื่อ field โดยไม่มีแผนย้ายข้อมูล

Presentation Layer — ชั้นการแสดงผล

ข้อมูลชุดเดียวถูกจัดเป็น 2 มุมมอง:

  • Portfolio: มีภาพ ผลงาน และ interaction
  • Resume: เน้น A4 อ่านง่าย พิมพ์ได้

Theme Layer — ชั้นอัตลักษณ์

Theme ไม่ใช่แค่เปลี่ยนสี แต่ควรเปลี่ยนบรรยากาศ เช่น Government สุภาพ, Minimal เรียบ, Esports กระฉับกระเฉงแต่ยังอ่านง่าย

Interaction Layer — ชั้นการโต้ตอบ

ประกอบด้วย cursor grid, card tilt, Coverflow และ mobile navigation โดยต้องลดเอฟเฟกต์ใน print และจอที่ต้องการ reduced motion

4. Coverflow ทำงานอย่างไร

แทนที่จะใช้แถวรูปยาว ๆ ที่ทำให้เกิดพื้นที่ว่าง ระบบใช้ progress ของการ scroll แนวตั้งเพื่อเลือกภาพที่ควรเด่น

6–12%เปิด
12–76%เลื่อนภาพ
76–90%ค้างภาพสุดท้าย
90–100%ปิด
openAfter = 0.06
startMove = 0.12
endMove = 0.76
collapseAfter = 0.90

ควรเกิด

  • 1 รูปแสดงปกติ
  • 2 รูปขึ้นไปใช้ Coverflow
  • Desktop/Mobile ใช้แนวคิดเดียวกัน
  • รูปสุดท้ายมีช่วงให้ดู
  • การ์ดถัดไปขึ้นต่อทันที

ไม่ควรเกิด

  • horizontal blank rail
  • พื้นที่ว่างยาวหลังรูปสุดท้าย
  • Mobile เปิดแต่ไม่ปิด
  • Print ซ่อนรูปเพราะสถานะ collapsed
  • มี engine ซ้ำควบคุม interaction เดียวกัน

5. Resume และ Theme

Resume ควรใช้ token จาก theme เดียวกับ Portfolio:

--resume-accent
--resume-accent-2
--resume-sidebar-1
--resume-sidebar-2
--resume-sidebar-3

หัวข้อ เส้น Timeline จุด Timeline skill chips และปุ่มแก้ไขไม่ควร hard-code เป็นสีเดิมเมื่อเปลี่ยน theme

6. Print และ Export

เมื่อพิมพ์ ระบบต้องเปลี่ยนจาก interactive เป็นเอกสาร:

  • ซ่อน nav, modal, cursor grid และปุ่ม Edit
  • เปิดเนื้อหาที่กำลัง collapsed
  • เปลี่ยนรูปเป็น grid/static
  • คง contrast และขนาด A4
  • WEB export ต้องซ่อน editor controls

ทำไมต้องเปลี่ยน cache version?

Browser อาจเก็บ CSS/JS เวอร์ชันเดิมไว้ แม้ deploy ใหม่แล้ว การเปลี่ยน ?v=... ช่วยบังคับให้ browser ดึงไฟล์ใหม่

ทำไมเห็นใน GitHub แต่เว็บไม่เปลี่ยน?

อาจเกิดจาก deploy ยังไม่เสร็จ, asset ไม่ถูก inject, dist ไม่ sync, cache ยังเก่า หรือ selector ไม่ตรงกับ DOM จริง

8. Definition of Done แบบคนทั่วไปตรวจได้

หัวข้อคำถามตรวจ
Desktopเปิดเว็บแล้วหน้าไม่ล้น เมนูใช้ได้ Coverflow เปิด–เลื่อน–ค้าง–ปิด
Mobileเมนูแตะง่าย ไม่มี page overflow และรูปไม่สร้าง blank rail
Resumeสีตรง theme และ A4 ไม่ถูกตัด
Printไม่มี nav/edit/grid ติดไป รูปและข้อความครบ
Deployproduction โหลด asset ใหม่จริง ไม่ใช่ดูเฉพาะโค้ดใน repo
Recoveryมี backup branch หรือ commit ที่ย้อนกลับได้

สรุปในประโยคเดียว

ระบบนี้รับข้อมูลหนึ่งชุด แล้วแปลงให้เป็น Portfolio แบบ interactive และ Resume แบบเอกสาร โดยใช้ Theme และ Export ร่วมกัน พร้อม deploy เป็น static web ได้
