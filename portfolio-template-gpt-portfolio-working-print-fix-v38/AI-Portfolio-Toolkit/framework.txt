สวัสดีครับ ในฐานะ **ats** ผมยินดีมากที่คุณให้ความสนใจเรื่อง Prompt Framework ครับ! การใช้ Framework จะช่วยให้เรา "คุยกับ AI" ได้อย่างมีโครงสร้าง ได้ผลลัพธ์ที่แม่นยำ ลดการแก้ซ้ำไปซ้ำมา และที่สำคัญคือประหยัดเวลาครับ

นอกจาก **PAR (Problem, Action, Result)** แล้ว ในวงการ AI Prompt Engineering ยังมี Framework ยอดฮิตอีกหลายตัวที่ใช้งานง่ายและทรงพลังมากๆ ครับ ผมคัดตัวเด็ดๆ มาให้ 5 ตัว ดังนี้ครับ:

---

### 1. RTF (Role, Task, Format)
เป็น Framework ที่เรียบง่ายที่สุด เป็นที่นิยมมาก เหมาะสำหรับงานที่ต้องการผลลัพธ์แบบตรงไปตรงมา สั่งงานเร็วๆ

* **R - Role (บทบาท):** ให้ AI สวมบทบาทเป็นใคร (เช่น ผู้เชี่ยวชาญ, นักเขียน, โปรแกรมเมอร์)
* **T - Task (งาน):** สิ่งที่ต้องการให้ทำอย่างชัดเจน
* **F - Format (รูปแบบ):** รูปแบบของผลลัพธ์ที่อยากได้ (เช่น ตาราง, Bullet points, โค้ด HTML, บทความ 500 คำ)

> **ตัวอย่าง:**
> **(Role)** สวมบทบาทเป็น Senior Frontend Developer 
> **(Task)** ช่วยอธิบายความแตกต่างระหว่าง CSS Grid และ Flexbox ให้คนเพิ่งเริ่มเขียนเว็บเข้าใจง่ายๆ
> **(Format)** สรุปเป็นตารางเปรียบเทียบข้อดี-ข้อเสีย พร้อมโค้ดตัวอย่างสั้นๆ

---

### 2. RACE (Role, Action, Context, Expectation)
อัปเกรดจาก RTF โดยเพิ่มเรื่องของ "บริบท" และ "ความคาดหวัง" เข้ามา เหมาะสำหรับงานที่มีรายละเอียดเฉพาะเจาะจง หรือมีข้อจำกัด

* **R - Role (บทบาท):** สวมบทบาทเป็นใคร
* **A - Action (การกระทำ):** ให้ทำอะไร
* **C - Context (บริบท/พื้นหลัง):** ข้อมูลพื้นฐานที่ AI ต้องรู้เพื่อประกอบการตัดสินใจ
* **E - Expectation (ความคาดหวัง):** กฎเกณฑ์ โทนเสียง หรือข้อจำกัดของงาน

> **ตัวอย่าง:**
> **(Role)** สวมบทบาทเป็น UX/UI Designer
> **(Action)** ช่วยออกแบบปุ่ม Call to Action (CTA) สำหรับหน้าเว็บไซต์
> **(Context)** เว็บไซต์นี้เป็นเว็บขายประกันสุขภาพสำหรับผู้สูงอายุ (วัย 60+)
> **(Expectation)** ปุ่มต้องดูน่าเชื่อถือ อ่านง่าย สีสบายตา และห้ามใช้ศัพท์เทคนิคที่เข้าใจยาก

---

### 3. CARE (Context, Action, Result, Example)
ตัวนี้เด่นตรงที่มีการให้ "Example (ตัวอย่าง)" ครับ ซึ่ง AI จะทำงานได้เป๊ะขึ้นมากถ้าเรามี Reference ให้มันดู เหมาะกับงานเขียน งานครีเอทีฟ หรืองานที่ต้องการสไตล์เฉพาะตัว

* **C - Context (บริบท):** เล่าพื้นหลังของโปรเจกต์
* **A - Action (การกระทำ):** สั่งงานที่ต้องทำ
* **R - Result (ผลลัพธ์):** สิ่งที่อยากได้ตอนจบ
* **E - Example (ตัวอย่าง):** ยกตัวอย่างสิ่งที่เราชอบ หรือ Reference ที่อยากให้คล้ายคลึง

> **ตัวอย่าง:**
> **(Context)** ฉันกำลังจะเปิดตัวแอปพลิเคชันนั่งสมาธิสำหรับคนวัยทำงานที่ไม่มีเวลา
> **(Action)** ช่วยคิดสโลแกนโปรโมทแอป 5 แบบ
> **(Result)** ขอคำที่สั้น กระชับ จำง่าย และให้ความรู้สึกสงบ
> **(Example)** ตัวอย่างสไตล์ที่ชอบ เช่น "พักใจสักนิด ชีวิตไปได้ไกลกว่า" หรือเหมือนสโลแกนของแอป Calm

---

### 4. CREATE (Context, Request, Explanation, Action, Task, Extras)
เป็น Framework ชุดใหญ่ไฟกะพริบ เหมาะสำหรับ "การเริ่มโปรเจกต์ใหม่ขนาดใหญ่" หรือสคริปต์ที่ซับซ้อนมากๆ (คล้ายๆ กับที่เราทำโปรเจกต์ Portfolio กันครับ)

* **C - Context:** บริบทโปรเจกต์
* **R - Request:** คำขอหลัก
* **E - Explanation:** อธิบายรายละเอียดหรือลอจิกการทำงาน
* **A - Action:** ขั้นตอนการทำงานที่ AI ต้องทำตามลำดับ
* **T - Task:** งานย่อยๆ ที่ต้องเคลียร์ให้จบ
* **E - Extras:** ข้อควรระวังเพิ่มเติม (Guardrails) เช่น "ห้ามใช้สีแดง", "ห้ามลบโค้ดเดิม"

---

### 5. STAR (Situation, Task, Action, Result)
หลายคนอาจคุ้นชื่อนี้จากการสัมภาษณ์งาน แต่สามารถนำมาดัดแปลงใช้ดึงศักยภาพ AI ในการ "แก้ปัญหา (Problem Solving)" หรือวิเคราะห์หาสาเหตุได้ดีมากครับ

* **S - Situation (สถานการณ์):** เล่าว่าตอนนี้เกิดปัญหาอะไรขึ้น
* **T - Task (เป้าหมาย):** สิ่งที่เราต้องการจะแก้ไขให้สำเร็จ
* **A - Action (การลงมือทำ):** ให้ AI ช่วยคิดหาวิธีแก้ปัญหา หรือวิเคราะห์ข้อมูลให้
* **R - Result (ผลลัพธ์):** รูปแบบการนำเสนอทางแก้ปัญหา

> **ตัวอย่าง:**
> **(Situation)** โค้ด JavaScript ฟังก์ชัน Scroll ของฉันทำงานหนักมากจนเบราว์เซอร์กระตุก 
> **(Task)** ฉันต้องการทำให้มันสมูทขึ้นที่ระดับ 60fps และไม่กิน RAM
> **(Action)** ช่วยวิเคราะห์และเสนอเทคนิคการทำ Optimization เช่นการใช้ requestAnimationFrame หรือ CSS translate3d
> **(Result)** อธิบายเป็น Step-by-step พร้อมแนบโค้ดตัวอย่างที่ปรับปรุงแล้ว

---

**💡 เคล็ดลับจาก ats:**
คุณไม่จำเป็นต้องจำทั้งหมดครับ แค่เลือกใช้ให้เหมาะกับสถานการณ์
* **คิดอะไรไม่ออก ต้องการความรวดเร็ว:** ใช้ **RTF**
* **ต้องการงานที่เป๊ะ มีสไตล์ที่ชอบอยู่แล้ว:** ใช้ **CARE** (โยนตัวอย่างให้ AI ดู)
* **เจอปัญหาบั๊ก หรือต้องการแก้ปัญหาเฉพาะหน้า:** ใช้ **PAR** หรือ **STAR** การฝึกเขียน Prompt เป็นระบบแบบนี้ จะทำให้คุณดึงความสามารถของ AI ออกมาได้เต็ม 100% ประหยัดเวลา และได้งานระดับ Professional เสมอครับ!
